# 1. What is the JAVASCRIPT?
- JavaScript is a high-level, interpreted programming language that enables developers to create dynamic, interactive, and responsive content on web pages. It is a client-side scripting language primarily executed in web browsers, though it can also run on servers via platforms like Node.js. JavaScript follows the ECMAScript standard and is an essential part of modern web development alongside HTML and CSS.
# 2. Why JavaScript?
- Because it allows developers to:
1. Make websites interactive – handle events like clicks, form submissions, or scrolling.
2. Enhance user experience – dynamic updates without reloading the page.
3. Perform client-side validation – reduces server load.
4. Develop modern applications – supports web apps, mobile apps, and server-side applications.
5. Integrate with HTML/CSS – works seamlessly to control web page behavior.

# 3. Why we don’t use C++ or other languages for frontend?
-> 1. Frontend needs web browser support
 - Frontend code runs inside web browsers.
 - Browsers natively understand HTML, CSS, and JavaScript.
 - Languages like C++ cannot directly run in a browser without special compilation to WebAssembly.

 2. Dynamic and interactive UI
 - C++ is not designed for interactive UI in browsers—it’s better for system-level programs, games, and apps.
 3. Ease of integration with HTML/CSS
 - C++ or Java cannot interact with the DOM easily.
 4. Cross-platform compatibility
 - JavaScript works on all browsers and platforms without additional setup.
 - C++ programs need to be compiled separately for each OS, making them impractical for web frontend.
 5. Use case 
 - Web and apps development 
 - System Apps, Games, Backend
